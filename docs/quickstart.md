# Quickstart — from clone to a nightly paper loop

> **What this gets you:** the current candidate book (B1 monthly cross-sectional
> momentum, docs/momentum_design.md) trading **paper money** on your own free
> Alpaca paper account, with every fill, NAV mark, and book-vs-target
> divergence recorded to append-only ledgers.
>
> **What this does not claim:** that you will make money. This repo's own
> evidence bar (SPEC.md §10) has never been cleared by any configuration —
> the first candidate was certified *uneconomic* and archived
> (docs/certifications/001) — and the momentum candidate's promotion verdict
> is unreadable before its pre-registered window (≥ 2027-06). Prism is a
> harness for finding out honestly, not a promise of alpha. Nothing here is
> investment advice; the license is MIT, no warranty.

## 1. Prerequisites

- Linux or WSL2 (the research extra is Linux-only; the core stack is what you
  need here), Python ≥ 3.12, [uv](https://docs.astral.sh/uv/).
- A free [Alpaca](https://alpaca.markets) account — you only need the **paper**
  API keys. Prism never touches bank credentials; funding and account linkage
  live at the broker (docs/security.md §1).
- Optional: a free [Twelve Data](https://twelvedata.com) API key (8 req/min,
  800/day) — the research spine's bar source. The live loop defaults to
  Alpaca's own IEX feed and runs without it.

```sh
git clone https://github.com/boom90lb/prism && cd prism
uv sync          # core + dev; add --extra research only on Linux, only for research scripts
```

## 2. Credentials

Create `.env` in the repo root (gitignored — see docs/security.md for the
doctrine: env-only, never logged, paper-first):

```
APCA_API_KEY_ID=<your paper key id>
APCA_API_SECRET_KEY=<your paper secret>
TWELVEDATA_API_KEY=<optional>
```

Leave `APCA_API_BASE_URL` unset: the paper endpoint is the default, and
pointing at the live endpoint is a deliberate act this quickstart does not
cover (see §7).

## 3. Preflight

```sh
uv run prism-doctor                # offline: env, universe, data dir, loop state, kill switch
uv run prism-doctor --network      # + live probes: Alpaca account (free), Twelve Data (1 credit)
```

Fix anything that FAILs; WARNs are runnable-but-look. Every first-run failure
class this catches otherwise appears as a mid-loop crash at 21:30.

## 4. Universe

The point-in-time S&P 500 membership file ships in-tree
(`data/universe/sp500_current.txt`, ~500 names). It is regenerated by the
maintainer's data program; you do not need to rebuild it to start. If you edit
it: a held name missing from the file is fetched for valuation/exit
automatically, but universe edits bite at the *next valuation*, immediately —
run `prism-doctor` after any edit.

## 5. One cycle by hand

Run once after the close (OPG next-open orders must reach Alpaca before
~09:28 ET next morning):

```sh
uv run python -m prism.scripts.paper_loop \
    --book momentum \
    --universe-file data/universe/sp500_current.txt \
    --run-dir runs/paper_loop_momentum
```

First run: fetches ~3y of bars for ~500 names from the Alpaca feed (minutes),
scores, constructs the decile long/short book, and submits next-open orders.
Re-running the same evening is safe — the write-ahead protocol resumes the
persisted decision and never double-trades (SPEC §7.7, N2).

Next morning after the open (~09:35 ET), sweep the auction's unexecuted
residuals (the paper venue prints only ~20–25% of OPG orders):

```sh
uv run python -m prism.scripts.paper_sweep --run-dir runs/paper_loop_momentum
```

## 6. Schedule it

Wrap the two commands in your scheduler of choice — cron, systemd timers, or
Windows Task Scheduler via `wsl.exe`. Pattern: an evening loop run (any time
between the close and ~09:00 ET) and a morning sweep (~09:35 ET), each
appending to a log in the run dir. Weekdays only; a holiday run no-ops on the
stale-panel guard.

What accumulates in `--run-dir`:

| File | What it is |
|---|---|
| `equity.jsonl` | one mark-to-market NAV row per session (the monitor's return stream) |
| `fills.jsonl` | every fill beside its decision-close reference price (arrival slippage) |
| `targets.jsonl` | the constructed book at each refresh (what the loop *should* hold) |
| `concordance.jsonl` | held-vs-target divergence per session (active share, gross, corr) |
| `unfilled.jsonl` | every residual the venue did not print |
| `state.json` | durable loop state — positions, cash, pending orders (never edit by hand) |

## 7. Safety rails and stopping

- **Stop the book:** `touch <run-dir>/KILL_SWITCH`. The loop still settles and
  marks NAV but submits nothing (and the sweep refuses too); delete the file
  to resume. `prism-doctor` reports a present kill switch loudly.
- **Drawdown halt:** the loop halts below 50% of NAV peak by default
  (`--max-drawdown`); tighten it if you care about the account.
- **Order guards:** per-order notional and order-count bounds derived from the
  construction caps veto a corrupted decision *before* anything is persisted
  or submitted.
- **Real money:** deliberately out of scope here. It requires explicitly
  setting `APCA_API_BASE_URL` to the live endpoint, and you should read
  SPEC.md §10 first — the repo's own doctrine is that no configuration has
  cleared the deployment evidence bar. Whole-share OPG sizing also sets a
  measured account-size floor: ~$100k holds the ~100-name decile book
  faithfully, $50k is borderline, and $10k trades a structurally different
  portfolio (40 of 98 names never enter — docs/account_size_floor.md). If
  you do it anyway, tighten every rail and treat the first weeks as a
  micro-account shakedown (SPEC amendment A3 precedent,
  docs/amendments_2026-07.md).
